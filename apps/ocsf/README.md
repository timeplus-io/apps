# OCSF Security Event Analytics

Real-time analytics over [OCSF](https://schema.ocsf.io/) events generated by
the [`timeplus-ocsf-simulator`](https://pypi.org/project/timeplus-ocsf-simulator/)
PyPI package.

## Install

From the repo root:

```bash
make build install APP=ocsf
```

Or build and install directly:

```bash
cd apps/ocsf && make build
curl -X POST http://localhost:8000/default/api/v1beta2/apps/install \
  -F "file=@ocsf.tpapp"
```

## What it builds

- `ocsf.ocsf_events_source` — Python external stream calling the simulator
- `ocsf.ocsf_events` — typed stream `(raw, class_uid)` with configurable TTL
- `ocsf.mv_ocsf_events_ingest` — MV from source → stream
- Four flatten views:
  - `v_ocsf_authentication_flatten` (3002)
  - `v_ocsf_network_activity_flatten` (4001)
  - `v_ocsf_process_activity_flatten` (1007)
  - `v_ocsf_security_finding_flatten` (2001)
- One dashboard, **OCSF Security Events**, with 17 panels grouped by event class

## Config

| Key | Default | Notes |
|---|---|---|
| `event_classes` | `1007,2001,3002,4001` | comma-separated OCSF class UIDs |
| `interval_seconds` | `0.1` | seconds between simulator events (~10 eps) |
| `ocsf_version` | `1.1.0` | OCSF schema version |
| `retention_hours` | `24` | TTL for the `ocsf_events` stream |

Override with the standard `config[<key>]=<value>` form fields at install time.

## Notes

- Simulator data is random per event — cross-class joins by IP or user will
  not correlate. Each dashboard section is self-contained per event class.
- The `timeplus-ocsf-simulator` Python package install is async; the
  external stream's Python generator has a reconnect loop that tolerates
  the initial race on a fresh install.
- All four flatten views are projected from the same `ocsf_events` stream
  using a `class_uid` integer filter, so they share TTL with the source.
