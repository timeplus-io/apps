# Alpha 101 Demo Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a `.tpapp` package `apps/alpha-101/` that simulates a 10-stock market feed and computes WorldQuant Alpha #1 as a streaming view.

**Architecture:** A `CREATE RANDOM STREAM` feeds an MV into a persistent `market_data` stream; a chain of views computes 1s bars → returns + 20-bar stddev + signed-power → 5-bar ts_argmax → cross-sectional rank − 0.5.

**Tech Stack:** Timeplus 3.3.1 (Proton), pure SQL with Go-template `{{ .DB }}` variables, no Python, ClickHouse-style HTTP on port 8123 for ad-hoc validation, REST app-install on port 8000.

**Reference:** `docs/superpowers/specs/2026-05-20-alpha-101-demo-design.md`

## Conventions used by every task

**Validation pattern.** Each DDL file uses two Go-template variables — `{{ .DB }}` (resolved to `alpha_101`) and `{{ .Config.bucket }}` (resolved to the user-selected bar interval, default `1s`). To validate against the running Timeplus we substitute both manually. Standard one-liner used in this plan (run from repo root):

```bash
sed -e "s/{{ \\.DB }}/alpha_101/g" -e "s/{{ \\.Config\\.bucket }}/1s/g" \
    apps/alpha-101/ddl/<FILE>.sql \
  | curl -s -u 'proton:proton@t+' --data-binary @- http://localhost:8123/
```

A blank response = success (the SQL executed and returned no rows). Non-empty output starting with `Code:` = error.

**Clean-slate helper.** Before re-running validation:

```bash
echo "DROP DATABASE IF EXISTS alpha_101" \
  | curl -s -u 'proton:proton@t+' --data-binary @- http://localhost:8123/
echo "CREATE DATABASE IF NOT EXISTS alpha_101" \
  | curl -s -u 'proton:proton@t+' --data-binary @- http://localhost:8123/
```

**Live-validated function notes** (probed against the running 3.3.1-rc.5 instance):

- Use `array_reduce('stddev_pop', arr)` — `array_stddev_pop` does **not** exist.
- `arr[i]` bracket indexing works (1-indexed).
- `array_first_index(lambda, arr)` works.
- `array_sort(x -> x.2, [(a,1),(b,3),(c,2)])` sorts tuples by 2nd element.
- `array_enumerate(arr)` returns `[1..length(arr)]`.
- `range(1, 21)` returns `[1..20]` (exclusive end).

### Task 5: Features view — returns, σ_ret_20, signed_power (DDL 005)

**Files:**
- Create: `apps/alpha-101/ddl/005_v_features.sql`

- [ ] **Step 1: Write the SQL**

```sql
CREATE OR REPLACE VIEW {{ .DB }}.v_features AS
SELECT
  time, stock_id, close, returns, sigma_ret_20,
  cond,
  sign(cond) * cond * cond AS signed_power
FROM (
  SELECT
    time, stock_id, close, returns, sigma_ret_20,
    if(returns < 0, sigma_ret_20, close) AS cond
  FROM (
    SELECT
      time, stock_id, close,
      (close21[1] - close21[2]) / nullif(close21[2], 0) AS returns,
      array_reduce(
        'stddev_pop',
        array_map(
          i -> (close21[i] - close21[i + 1]) / nullif(close21[i + 1], 0),
          range(1, 21)
        )
      ) AS sigma_ret_20
    FROM (
      SELECT
        time, stock_id, close,
        lags(close, 0, 20) AS close21
      FROM {{ .DB }}.v_bars
      PARTITION BY stock_id
    )
  )
)
```

### Task 7: Cross-sectional rank − 0.5 (DDL 007)

**Files:**
- Create: `apps/alpha-101/ddl/007_v_alpha_1.sql`

- [ ] **Step 1: Write the SQL**

```sql
CREATE OR REPLACE VIEW {{ .DB }}.v_alpha_1 AS
WITH ranked AS (
  SELECT
    window_start AS time,
    array_sort(p -> p.2, group_array((stock_id, ts_argmax))) AS sorted_pairs,
    length(group_array(stock_id))                            AS n
  FROM tumble({{ .DB }}.v_ts_argmax_5, {{ .Config.bucket }})
  WHERE ts_argmax IS NOT NULL
  GROUP BY window_start
)
SELECT
  time,
  sorted_pairs[idx].1                        AS stock_id,
  cast(idx, 'float64') / n - 0.5             AS alpha_1
FROM ranked
ARRAY JOIN array_enumerate(sorted_pairs) AS idx
```

- [ ] **Step 2: Validate compile**

```bash
sed -e "s/{{ \\.DB }}/alpha_101/g" -e "s/{{ \\.Config\\.bucket }}/1s/g" \
    apps/alpha-101/ddl/007_v_alpha_1.sql \
  | curl -s -u 'proton:proton@t+' --data-binary @- http://localhost:8123/
```

Expected: empty. If `ARRAY JOIN array_enumerate(...)` fails inside a `WITH`, fall back to wrapping the CTE in a subquery (drop the `WITH`):

```sql
SELECT
  time,
  sorted_pairs[idx].1 AS stock_id,
  cast(idx, 'float64') / n - 0.5 AS alpha_1
FROM (
  SELECT
    window_start AS time,
    array_sort(p -> p.2, group_array((stock_id, ts_argmax))) AS sorted_pairs,
    length(group_array(stock_id)) AS n
  FROM tumble({{ .DB }}.v_ts_argmax_5, {{ .Config.bucket }})
  WHERE ts_argmax IS NOT NULL
  GROUP BY window_start
)
ARRAY JOIN array_enumerate(sorted_pairs) AS idx
```
